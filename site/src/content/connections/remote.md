## How It Works

ha-mcp is exposed to the internet via a secure HTTPS tunnel or reverse proxy.

```
┌─────────────────┐
│   AI Client     │
│  (Anywhere)     │
└────────┬────────┘
         │ HTTPS
         ↓
┌─────────────────────────────────────────────┐
│            Secure Tunnel / Proxy            │
│  (Cloudflare Tunnel, Caddy, Nginx, etc.)    │
└────────────────────┬────────────────────────┘
                     │ HTTP
                     ↓
         ┌─────────────────────┐
         │      ha-mcp         │
         │   HTTP Server       │
         └──────────┬──────────┘
                    │
                    ↓
         ┌─────────────────────┐
         │   Home Assistant    │
         └─────────────────────┘
```

## Requirements

- ha-mcp running as HTTP server
- Reverse proxy or tunnel for HTTPS
- Web clients (ChatGPT) **require HTTPS**

## Best For

- Web-based AI clients (ChatGPT, Claude.ai)
- Accessing from outside your network
- Mobile or travel usage

## Security Considerations

- **Secret path**: Use a random secret in URL (auto-generated by HA Add-on)
- **HTTPS required**: Never expose HTTP directly to internet
- **Zero Trust** (optional): Use Cloudflare Access for additional auth

## Deployment Options

1. **Cloudflare Tunnel** (Recommended)
   - No port forwarding needed
   - Free tier available
   - Works behind CGNAT

2. **Caddy Reverse Proxy**
   - Automatic HTTPS via Let's Encrypt
   - Requires public IP or dynamic DNS

3. **Nginx + Let's Encrypt**
   - Traditional setup
   - Full control over configuration
